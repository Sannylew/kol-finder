# 更新日志

本项目版本变更记录。遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [1.3.1] - 2026-07-07

### 新增
- **后台版本更新**：系统菜单新增「版本更新」页面，可检查 GitHub Release 最新版本，并在支持的 Linux/systemd 部署环境中通过固定脚本启动后台更新任务。
- **仪表盘更新提醒**：仪表盘检测到新版本时显示更新提示；更新任务运行中会显示状态提示并可跳转查看。

### 变更
- **排序文案统一**：后台和文档中将「优先级」统一调整为「展示排序 / 前台展示排序」，保留数字越低越靠前的规则。

### 安全与稳定性
- 更新接口仅管理员可访问；不接受自定义命令，仅启动项目内固定 `scripts/update.sh`。
- 更新任务使用 `systemd-run` 托管，避免后端重启时中断任务；本地 Windows 或非 systemd 环境会显示不支持，避免误触发。
- 防止重复启动更新任务，并校验目标版本号格式。

## [1.2.0] - 2026-07-03

### 新增
- **包裹图片**：博主详情新增包裹图功能，可上传多张（每人上限 20 张），缩略图网格展示，点击放大查看（灯箱，支持左右切换 / Esc 关闭），支持逐张删除；访客可见（脱敏时也照常展示），上传/删除入口仅管理员可见
- **独立后台控制台**：后台管理从弹窗重构为独立页面 `/admin`（登录后访问），左侧菜单 + 顶部条布局，移动端菜单可收起；含仪表盘、博主列表、已移除博主、数据源设置、同步记录、外观设置、系统日志、数据备份、修改密码
- **仪表盘**：概览博主总数、今日/本周新增、已签/未签、已配照片、最近同步、待清理数，并可一键同步
- **同步记录页**：查看历史同步日志（时间/新增/更新/结果）
- **博主展示排序**：可调整博主前台展示顺序（数字越低越靠前），支持拖动、上移/下移、置顶和移到默认排序；本地附加，同步不覆盖
- **文档删除识别与清理**：同步自动识别文档中已删除的博主，右上角（后台）入口展示待清理数量，可逐个或一键清理（连带主图、包裹图）
- **公司名称展示**：后台可配置公司名称，显示在前台左上角品牌区（醒目徽标，所有访问者可见）
- **卡片公司显示开关**：后台可控制前台卡片是否显示公司行（默认隐藏，详情页与后台不受影响）
- **浏览体验优化**：返回顶部按钮、每页数量选择（20/50/100，记住偏好）、一键重置筛选、浏览状态同步到 URL（刷新/分享/前进后退保留筛选与页码）
- 前端页脚显示当前版本号（后端新增 `/api/version`）

### 变更
- **前台展示逻辑**：文档已删除的博主不再显示在前台卡片墙、统计与筛选项中，仅在后台「已移除博主」列表待清理
- 博主列表默认按展示排序（已排序在前、其余按序号），排序结果稳定确定
- **取消默认密码强制修改**：默认密码可正常登录，改为后台内温和提示「建议修改密码」（不再拦截）

### 修复
- 删除博主改为单事务删除数据库记录、提交后统一清理文件，避免 DB 与磁盘不一致
- 修复带 `page` 参数的链接刷新后跳回第一页的问题
- 清理若干冗余代码（重复导入、死导入）

### 说明
- 新增字段 `in_doc`、`priority` 与表 `kol_package_photo`，旧库启动自动补列/建表，无需手动迁移
- 新增前端依赖 `react-router-dom`，部署时 `update.sh` 会自动安装
- 「只读金山文档」原则不变；本次为 SQLite→SQLite 普通更新，服务器用 `scripts/update.sh` 部署即可

## [1.1.0] - 2026-07-01

### 变更
- **数据库从 PostgreSQL 改为 SQLite**（单文件，零配置），移除 Docker 依赖，大幅简化部署
- 备份/恢复改为直接操作 SQLite 文件（`.db.gz`），部署脚本精简为两步
- nginx 配置默认开启 gzip 压缩与静态资源缓存

### 修复
- upsert 改用版本无关写法，修复不同 SQLAlchemy 版本下 `on_conflict_do_update` 兼容报错
- 迁移脚本自动将 `.env` 的 `DATABASE_URL` 从 PostgreSQL 切换为 SQLite（否则停 PG 后后端崩溃）
- 迁移脚本在切换数据库后再导入照片，避免照片关联写入错误的库
- 备份文件名同秒冲突（自动加序号避免覆盖）
- install/update/migrate 脚本在 GitHub 不稳时自动回退国内镜像

### 新增
- `backend/migrate_photos.py`：从旧版 PostgreSQL 迁移照片关联到 SQLite
- `scripts/migrate_to_sqlite.sh`：一键自动迁移（备份→导出照片映射→切库→导入照片→重建前端）

### 说明
- 旧版（PostgreSQL）升级：博主数据从金山重新同步，照片自动迁移保留（详见 README）

## [1.0.2] - 2026-06-30

### 安全
- 移除仓库中预设的真实数据库密码与 `AUTH_SECRET`，改为占位符 `__AUTO_GENERATE__`
- 部署脚本 `deploy.sh` 首次部署时自动生成随机数据库密码与 `AUTH_SECRET`

### 新增
- `scripts/update.sh`：从 GitHub 拉取最新代码并自动更新（git pull + 重装依赖 + 重启 + 重建前端）
- `VERSION` 文件：版本号单一来源
- `CHANGELOG.md`：变更记录

### 变更
- 更新方式改为基于 git 拉取（详见 README「从 GitHub 更新」）
